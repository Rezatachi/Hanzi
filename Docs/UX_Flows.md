# UX Flows

## First Launch
1. Loading splash
2. Onboarding flow
3. Reminder preference
4. First card preview
5. Today tab

## Study Session
1. User starts Review/New/Saved session
2. Due queue loads overdue, learning, review, then new cards
3. Card reveals
4. User grades Again/Hard/Good/Easy
5. SRS updates and widget state refreshes

## Search
1. Query by Hanzi, pinyin, or English
2. Filter by saved/due/learned
3. Open detail
4. Save or review

## Widgets
1. User chooses widget mode in settings
2. App writes compact shared widget JSON
3. Widget reads shared state
4. Deep link opens Today, review, or a card detail route

## Wallpaper
1. User chooses style and card count
2. App renders portrait image
3. User shares or saves manually
4. App does not claim automatic wallpaper setting
